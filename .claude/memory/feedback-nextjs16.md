**Rule:** The FleetGuard project actually runs **Next.js 16.2.3** (see `package.json`). The Delivery Brief text mentions Next.js 14, but the installed toolchain is 16 with Turbopack — follow 16 conventions.

**Why:** AGENTS.md at project root states *"This is NOT the Next.js you know. This version has breaking changes — APIs, conventions, and file structure may all differ from your training data."* That warning is authoritative for this codebase.

**How to apply:**
- **Read the local docs** at `node_modules/next/dist/docs/` before writing new page/layout/route code — it's the ground truth for this exact version
- Dynamic route `params` is now `Promise<{…}>` — must be awaited: `const { token } = await params`
- `PageProps<'/path'>` / `LayoutProps<'/path'>` are globally-available helpers (generated by `next typegen`)
- Tailwind is v4 (`@import "tailwindcss"` + `@theme {…}` tokens in CSS) — there is no `tailwind.config.js`
- **Server → client component boundary is strict**: passing function references (e.g. lucide icon component refs) from a server layout into a client component crashes at prerender. The workaround used in this project: the Sidebar has an internal string-keyed icon registry; layouts pass `icon: "home"` not `icon: Home`. Do NOT undo this pattern.
- CSS files must not contain stray non-CSS characters — Turbopack's PostCSS pipeline chokes on them (once hit this by accidentally leaving XML fragments from a tool call at the end of `globals.css`)
- Build command is `npx next build` — produces Turbopack output; takes ~8s clean
