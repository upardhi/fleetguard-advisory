# Security Policy

## Supported versions

Only the latest commit on `main` is actively supported. Hotfixes are
backported only to releases less than 90 days old.

## Reporting a vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email **security@fleetguard.in** with:

1. A clear description of the vulnerability and its impact.
2. Steps to reproduce or a proof-of-concept (if safe to share).
3. Affected component(s) and version/commit hash.

We will acknowledge within **48 hours** and aim to release a fix within
**14 calendar days** for critical severity. We follow responsible disclosure
and will coordinate a public advisory with you.

## Security controls summary

See [`docs/SECURITY_CONTROLS.md`](docs/SECURITY_CONTROLS.md) for the full
technical control inventory (aligned to SOC 2 CC series and ISO 27001 Annex A).

## Bug bounty

No formal programme at this time. We do credit reporters in release notes
unless they prefer anonymity.
